## 문제 1: 유저 서비스 재방문(재결제) 기간(Time Delta) 구하기
**목표:** 각 유저별로 현재 결제일과 직전 결제일 간의 차이(일수)를 계산해야 합니다.

**힌트:**
1. **정렬하기 (`sort_values`)**: '이전 데이터'와 비교하려면 데이터가 먼저 시간순으로 나열되어 있어야 합니다. 유저 ID와 날짜를 기준으로 정렬하는 것을 잊지 마세요.
   - 키워드: `.sort_values(by=[...])`
2. **이전 데이터 구하기 (`shift`)**: 유저별로 데이터를 묶은 뒤, 날짜를 한 칸 아래로 밀어내면 같은 행에서 '현재 결제일'과 '이전 결제일'을 나란히 두고 뺄셈할 수 있습니다.
   - 키워드: `.groupby()`, `.shift()`
   ```python
   # 원리: 특정 그룹별로 값을 'n'칸 밀어내는 방법
   grouped_data.shift(n)
   ```
3. **날짜 차이 계산 후 정수화 (`dt.days`)**: Pandas에서 날짜(datetime) 끼리 빼기를 하면 `timedelta` 자료형이 만들어집니다. 여기서 순수하게 'n일'이라는 정수만 빼내려면 `.dt.days` 속성을 사용합니다.
   ```python
   # 원리: 두 날짜의 차이에서 정수(일수)만 추출
   (날짜A - 날짜B).dt.days
   ```

## 문제 2: 이동 평균(Rolling Average)을 활용한 이상치(Anomaly) 탐지
**목표:** 최근 3분의 트래픽 평균을 구하고, 현재 트래픽이 그 2배를 엄격히 초과하는지 확인합니다.

**힌트:**
1. **최근 N개의 기준 평균 구하기 (`rolling`)**: 지정한 윈도우 크기(연속된 N개의 행) 단위로 데이터를 묶어가며 통계를 낼 수 있습니다.
   ```python
   # 원리: 연속된 N개 데이터에 대한 평균값 연산
   df['대상_컬럼'].rolling(window=N).mean()
   ```
2. **조건 비교하기**: 위에서 구한 이동 평균의 2배 값보다 현재 트래픽 값이 더 큰지(>) 비교합니다. Pandas에서는 조건식을 쓰면 결과가 `True` / `False`로 나오며, 이를 그대로 새로운 컬럼 값으로 넣으면 됩니다.

## 문제 3: 코호트(Cohort) 분석 기틀 만들기
**목표:** 유저별 '최초 결제일'을 구하고, 이를 이용해 가입월(코호트 기준월)과 주문월을 추출합니다.

**힌트:**
1. **유저별 최초 결제일 원본 형태로 붙이기 (`transform`)**: 평범하게 `groupby('유저').min()`을 쓰면 결과 행 개수가 유저 수만큼 줄어듭니다. 각 주문 건(원본 행) 옆에 해당 유저의 가입일(최솟값)을 그대로 붙여주려면 `transform`이 필요합니다.
   ```python
   # 원리: 그룹별 집계값을 원본 데이터의 행 개수 모양대로 반환
   df.groupby(['기준_컬럼'])['대상_컬럼'].transform('min')
   ```
2. **'년-월' 주기로 단순화하기 (`to_period`)**: 날짜 데이터에서 '일/시/분/초'를 떼어내고, 특정 주기(예: Month를 의미하는 `'M'`) 단위로만 묶는 기능입니다. 문자열 변환(`strftime`) 방식으로도 가능합니다.
   ```python
   # 원리: 날짜를 월 단위 등으로 단순화
   df['날짜_컬럼'].dt.to_period('M')
   ```
